# Aether_island v0.5 GitHub Heatmap 可视网格补齐记录

## 问题

上一版 v0.5 只实现了 GitHub Heatmap 的静态摘要：

```text
142 contributions · 31 active days · 9 streak
```

这不是完整意义上的“热力图”。真正的 heatmap 至少需要贡献格子数据，并在前端渲染为可见网格。

## 本次补齐

### Rust 数据契约

`WidgetSnapshot` 新增可选字段：

```rust
pub details: Option<WidgetDetails>
```

当前支持：

```rust
pub enum WidgetDetails {
    GithubHeatmap { days: Vec<GithubContributionDay> },
}
```

贡献日结构：

```rust
pub struct GithubContributionDay {
    pub date: String,
    pub count: u16,
    pub level: u8,
}
```

设计原则：

- `details` 是可选字段，普通 widget 不受影响。
- GitHub 特有数据放入 `WidgetDetails::GithubHeatmap`。
- 不把 GitHub 特例硬塞进通用字段。

### Rust 静态数据

当前生成 35 个静态贡献格子：

```text
5 行 x 7 列
```

贡献等级：

```text
0 = 无贡献
1 = 低贡献
2 = 中低贡献
3 = 中高贡献
4 = 高贡献
```

### Frontend 类型守卫

TypeScript 新增：

```ts
type GithubHeatmapDetails = {
  kind: "github-heatmap";
  days: GithubContributionDay[];
};
```

并通过 type guard 校验 Rust 传来的 payload，避免 UI 直接信任外部数据。

### UI 渲染

新增 DOM 容器：

```html
<div id="widget-detail" class="island__detail" aria-live="polite"></div>
```

当 snapshot 是 GitHub Heatmap 时，前端渲染：

```text
github-heatmap
└── github-heatmap__cell × 35
```

普通 widget 会自动清空详情区域。

## 防屎山措施

- 通用 snapshot 只新增 `details` 扩展点，不污染 `title/subtitle/tone`。
- GitHub 细节通过 tagged enum 管理，后续可以加 `Pomodoro`、`ScriptMonitor` 等 details。
- 前端通过 `isGithubHeatmapDetails()` 分支渲染，避免散落多个 `widget_id === "github-heatmap"` 判断。
- CSS 使用独立 `.github-heatmap` 命名空间，避免污染岛主体样式。

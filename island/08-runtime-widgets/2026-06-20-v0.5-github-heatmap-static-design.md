# Aether_island v0.5 GitHub Heatmap 静态 Widget 设计草案

## 结论

v0.5 不建议直接在现有 `core/widget.rs` 里追加 GitHub 逻辑。

更稳的做法是先将 widget 系统拆成目录模块，再新增 `github_heatmap.rs`。这样 GitHub 热力图只是一个独立 widget，不会污染 clock、mock、sidecar，也不会让 provider 承担业务细节。

## 目标边界

v0.5 只做静态数据版 GitHub Heatmap widget：

- Rust 端注册 `github-heatmap` widget。
- 静态贡献数据在 Rust widget 内生成或由独立 fixture 函数提供。
- 前端暂时沿用现有 `WidgetSnapshot` 字段展示摘要。
- 不接真实 GitHub API。
- 不接 Python sidecar。
- 不改 IPC 大协议。

## Rust 接口草案

### widget/mod.rs

```rust

pub mod mock;
pub mod sidecar_widget;
pub mod system_clock;

use std::time::Duration;

use super::{events::ShellEvent, snapshot::WidgetSnapshot};

pub type WidgetResult<T> = Result<T, WidgetError>;

#[derive(Clone, Debug, Eq, PartialEq)]
pub struct WidgetError {
    pub message: String,
}

pub trait IslandWidget: Send {
    fn id(&self) -> &'static str;
    fn title(&self) -> &'static str;
    fn priority(&self) -> u8;
    fn refresh_interval(&self) -> Duration;
    fn refresh(&mut self) -> WidgetResult<WidgetSnapshot>;

    fn handle_shell_event(&mut self, _event: &ShellEvent) -> WidgetResult<()> {
        Ok(())
    }

    fn shutdown(&mut self) -> WidgetResult<()> {
        Ok(())
    }
}

pub use github_heatmap::create_github_heatmap_widget;
pub use mock::create_mock_widget;
pub use sidecar_widget::create_sidecar_widget;
pub use system_clock::create_system_clock_widget;
```

### widget/github_heatmap.rs

```rust
use std::time::Duration;

use crate::core::snapshot::{WidgetSnapshot, WidgetTone};

use super::{IslandWidget, WidgetResult};

const GITHUB_HEATMAP_WIDGET_ID: &str = "github-heatmap";
const GITHUB_HEATMAP_TITLE: &str = "GitHub";
const GITHUB_HEATMAP_PRIORITY: u8 = 40;
const STATIC_REFRESH_INTERVAL_SECS: u64 = 300;

pub struct GithubHeatmapWidget {
    summary: GithubHeatmapSummary,
}

#[derive(Clone, Debug, Eq, PartialEq)]
pub struct GithubHeatmapSummary {
    pub total_contributions: u32,
    pub active_days: u16,
    pub longest_streak_days: u16,
}

impl IslandWidget for GithubHeatmapWidget {
    fn id(&self) -> &'static str {
        GITHUB_HEATMAP_WIDGET_ID
    }

    fn title(&self) -> &'static str {
        GITHUB_HEATMAP_TITLE
    }

    fn priority(&self) -> u8 {
        GITHUB_HEATMAP_PRIORITY
    }

    fn refresh_interval(&self) -> Duration {
        Duration::from_secs(STATIC_REFRESH_INTERVAL_SECS)
    }

    fn refresh(&mut self) -> WidgetResult<WidgetSnapshot> {
        Ok(self.snapshot())
    }
}

impl GithubHeatmapWidget {
    pub fn new(summary: GithubHeatmapSummary) -> Self {
        Self { summary }
    }

    fn snapshot(&self) -> WidgetSnapshot {
        WidgetSnapshot::with_tone(
            self.id(),
            self.title(),
            format_github_summary(&self.summary),
            WidgetTone::Ready,
            self.priority(),
        )
    }
}

pub fn create_github_heatmap_widget() -> Box<dyn IslandWidget> {
    Box::new(GithubHeatmapWidget::new(static_heatmap_summary()))
}

fn static_heatmap_summary() -> GithubHeatmapSummary {
    GithubHeatmapSummary {
        total_contributions: 142,
        active_days: 31,
        longest_streak_days: 9,
    }
}

fn format_github_summary(summary: &GithubHeatmapSummary) -> String {
    format!(
        "{} contributions · {} active days · {} streak",
        summary.total_contributions,
        summary.active_days,
        summary.longest_streak_days,
    )
}
```

## Provider 注册方式

```rust
use super::widget::{
    create_github_heatmap_widget,
    create_mock_widget,
    create_sidecar_widget,
    create_system_clock_widget,
    IslandWidget,
    WidgetError,
};

impl NativeWidgetProvider {
    pub fn new() -> Self {
        let mut provider = Self { widgets: Vec::new() };

        provider.register(create_system_clock_widget());
        provider.register(create_github_heatmap_widget());
        provider.register(create_mock_widget());
        provider.register(create_sidecar_widget());

        provider
    }
}
```

## v0.6 预留方向

v0.6 如果接真实 GitHub API，不要让 `GithubHeatmapWidget` 自己直接发网络请求。

推荐组合式接口：

```rust
pub trait GithubContributionSource: Send {
    fn load_summary(&mut self) -> WidgetResult<GithubHeatmapSummary>;
}
```

然后：

```rust
pub struct GithubHeatmapWidget<S>
where
    S: GithubContributionSource,
{
    source: S,
}
```

这样未来可以替换：

- `StaticGithubContributionSource`
- `PythonSidecarGithubContributionSource`
- `HttpGithubContributionSource`
- `CacheFirstGithubContributionSource`

而不需要重写 widget 主体。

## 防屎山措施

- GitHub widget 独立文件，避免继续膨胀 `widget.rs`。
- 静态数据生成、摘要格式化、snapshot 构造分离。
- provider 只负责注册 widget，不知道 GitHub 内部数据结构。
- v0.5 不引入网络请求，避免把 API、缓存、认证、UI 展示一次性混在一起。
- v0.6 预留 source trait，使用组合而不是继承。

# Tauri 初始化时包管理器怎么选

## 结论

在当前环境下，优先选：

```text
npm
```

原因很简单：

- 你已经确认 `node` 和 `npm` 都可用。
- `npm` 是最直接、最少额外工具链的一项。
- 对于 IslandGate 这种先把骨架跑起来的项目，先用最稳的默认方案最合适。

## 当前场景

脚手架正在询问：

```text
? Choose your package manager ›
```

可选项包括：

- `pnpm`
- `yarn`
- `npm`
- `deno`
- `bun`

对你现在这个阶段，不建议为了“看起来更现代”去切到 `pnpm`、`bun` 或 `deno`。

## 为什么选 npm

### 1. 环境已经准备好了

你当前机器已经有：

- `node v24.15.0`
- `npm 11.12.1`

这意味着无需额外安装别的包管理器。

### 2. 工具链最少

脚手架初始化的第一目标不是“最潮”，而是：

```text
少变量、少故障点、先跑起来
```

`npm` 足够满足这个目标。

### 3. 适合先做骨架

IslandGate 当前还在初始化阶段，优先级是：

- Tauri 壳子先起来。
- Rust 入口先确认。
- 前端 UI 先能显示。
- 之后再谈依赖优化。

## 什么时候再考虑 pnpm

如果后面前端依赖开始增多、你想更省磁盘空间或更快的依赖安装速度，可以再迁移到 `pnpm`。

但那是后话，不是现在初始化阶段的必要动作。

## 推荐操作

当前直接选：

```text
npm
```

然后继续完成初始化。

## 参考判断标准

初始化阶段的包管理器选择优先级：

1. 已安装且可用。
2. 最少额外配置。
3. 最少迁移成本。
4. 最少对项目初始阶段的干扰。

按这个标准，`npm` 最稳。

# Aether_island 动画丝滑度优化笔记

## 结论

想让 Aether_island 的动态效果更丝滑，优先优化的不是“更高帧率”本身，而是：

```text
减少布局重排
减少重绘面积
尽量只动 transform / opacity
把状态变化放进稳定的动画管线
```

如果动画的形态变化比较大，视觉上更顺滑的关键通常是“少改宽高，多改缩放/位移/透明度”。

## 先定义目标

对当前灵动岛壳来说，比较实际的体验目标是：

- hover 触发时没有卡顿感
- 展开和收起的速度一致
- 文字和状态点不会闪烁
- 鼠标反复进出时不抖动
- 窗口动画和内容动画尽量同步

## 真正的瓶颈通常在哪里

对这种前端壳来说，卡顿常见来源不是 CPU 算不动，而是：

- 频繁修改 `width` / `height`
- 频繁触发布局重排
- 复杂阴影和高模糊 `backdrop-filter`
- 页面里同时做太多 DOM 变化
- 事件触发太密，状态来回抖动

## 优先级最高的优化

### 1. 让动画尽量只动 `transform` 和 `opacity`

比起直接改宽高，更推荐：

```css
transform: scale(1.02);
opacity: 0.95;
```

原因：

- `transform` 通常走合成层，成本低
- `width` / `height` 更容易触发布局和重排

如果你当前想做“胶囊变大”，可以考虑把布局尺寸固定，内容层做缩放，而不是每次都真的改盒子尺寸。

### 2. 降低阴影和模糊强度

这些会比较吃合成：

- `box-shadow`
- `backdrop-filter: blur(...)`
- 大面积半透明叠层

如果已经足够像样，就别继续加重阴影。

### 3. 鼠标状态变化要防抖

如果鼠标快速进出导致状态来回切换，视觉会抖。可以让状态变化更稳一点：

- `mouseenter` 后进入 hover
- `mouseleave` 后稍微延迟再收起
- 如果再次进入，取消收起

这能减少“刚展开又收起”的抽搐感。

## 对当前项目最建议的做法

### 方案 A：保留宽高变化，但优化触发

这是最容易理解的做法：

- 保留 `island--hovered` 改宽高
- 把阴影减轻
- 背景尽量简单
- 内容层不要频繁重排

适合现在刚开始做样式。

### 方案 B：固定外壳尺寸，内部做动画

更顺滑的做法：

- 外壳尺寸保持稳定
- 通过 `transform`、`opacity`、`translateY` 做展开感
- 文字和副标题逐步出现

这通常比直接改盒子尺寸更稳。

## requestAnimationFrame 的作用

如果你后面要做更复杂状态动画，可以把状态更新放进 `requestAnimationFrame`：

```ts
requestAnimationFrame(() => {
  // 在下一帧统一更新 class / style
});
```

它的意义不是“强行提帧率”，而是：

- 把一堆变化合并到同一帧
- 避免连续小改动分散到多个渲染周期
- 让视觉更新更整齐

## 一套实用的动画规则

```text
1. 先固定布局，再做动效
2. 优先动 transform / opacity
3. 少动 width / height / margin / top / left
4. 阴影和 blur 保守使用
5. 鼠标状态切换要平滑，不要抖
6. 不要让前端逻辑在 hover 时做重计算
```

## 如果你想继续往上提丝滑度

下一步可以考虑：

- 把 hover 展开改成 transform 缩放
- 给文字出现加轻微延迟
- 减少 blur 强度
- 用更小的 DOM 结构
- 用更少的状态类切换

## 简历与面试表达

可以这样说：

```text
针对桌面常驻浮层，我将动画优化重点放在减少重排和重绘，优先使用 transform 与 opacity 驱动状态过渡，并通过 requestAnimationFrame 合并 UI 状态更新，从而降低 hover 展开时的视觉抖动和主线程压力。
```

# v0.4 Quality Gate

日期：2026-06-19

## 结论

本轮为 Aether_island 增加了统一质量门禁：

```powershell
pnpm check
```

它会一次性检查 Rust、前端和 Python sidecar 三层。当前已经运行通过。

这一步让 v0.4 更接近“预发布可验收”状态：以后不用靠记忆手动跑一串命令，直接执行一个入口即可复查项目健康度。

## 新增文件

```text
D:\Aether_island\scripts\check.ps1
```

脚本使用 PowerShell，符合当前 Windows-first 项目约束。

## package.json 更新

新增：

```json
{
  "scripts": {
    "check": "powershell -NoProfile -ExecutionPolicy Bypass -File ./scripts/check.ps1"
  }
}
```

使用方式：

```powershell
pnpm check
```

## 门禁内容

`pnpm check` 当前执行：

```powershell
cargo fmt --check --manifest-path ".\src-tauri\Cargo.toml"
cargo test --manifest-path ".\src-tauri\Cargo.toml"
cargo clippy --manifest-path ".\src-tauri\Cargo.toml" -- -D warnings
pnpm build
uv run ruff format .
uv run ruff check .
uv run pytest
```

其中 Python 命令会在 `island-daemon` 目录中执行。

如果未来某个分支暂时没有 `island-daemon`，脚本会跳过 Python daemon 检查，并给出提示。

## 对 v0.4 的意义

v0.4 已经横跨：

- Rust/Tauri shell。
- Win32 原生窗口控制。
- TypeScript 前端。
- Python sidecar 协议草图。

如果没有统一门禁，很容易出现某一层通过、另一层悄悄坏掉。

`pnpm check` 现在就是 v0.4 的基础健康证明。

## 后续建议

下一步可以把 `pnpm check` 当作所有提交前的本地门禁。等项目准备公开或发布时，再考虑把同样逻辑迁移到 GitHub Actions。

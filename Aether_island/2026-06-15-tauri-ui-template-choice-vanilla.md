# Tauri UI 模板为什么选 Vanilla

## 结论

在 Aether_island 这种目标明确的桌面灵动岛项目里，优先选：

```text
Vanilla
```

也就是不先引入 Vue、React、Svelte、Solid、Angular、Preact 这类前端框架。

## 原因

### 1. 先把壳子做轻

你的项目第一阶段重点不是复杂网页应用，而是：

- 顶部灵动岛窗口。
- 流体动画。
- 托盘。
- 鼠标穿透。
- Rust 后端调度。

这些用原生 HTML / CSS / JS 就足够。

### 2. 少一层框架抽象

前端框架会带来：

- 更多工程结构。
- 更多状态管理习惯。
- 更多构建约定。
- 更多依赖。

对于现在的目标，这些都不是刚需。

### 3. 更适合和 Rust 后端配合

IslandGate 这类项目里，前端只需要消费状态快照，Rust 负责逻辑和数据。

Vanilla 更容易保持这种简单边界：

```text
前端 = 展示
Rust = 数据和系统能力
```

## 什么时候再考虑框架

如果后面页面复杂了，比如：

- 多个面板。
- 复杂设置页。
- 大量交互表单。
- 很多共享状态。

那再考虑 Vue 或 Svelte 之类框架会更自然。

但现在不是那个阶段。

## 一句话建议

> 现在选 Vanilla，先把轻量骨架跑通；以后真需要更复杂的 UI，再换框架也不迟。
